WAFPass

WAFPass ist das offizielle CLI für das WAF++ Framework — ein automatisierter Compliance-Checker, der deine Terraform-Infrastruktur gegen die sieben Säulen prüft: Security, Cost, Performance, Reliability, Operations, Sustainability und Sovereignty.

Dokumentation

Vier Perspektiven. Eine konsistente Grundlage.

WAFPass wendet die vier PASS-Perspektiven als automatisierte Checks auf deine Infrastruktur an — und macht Compliance-Entscheidungen nachvollziehbar, wiederholbar und auditierbar.

Plattform

Validiert Basis-Plattform-Controls — Tagging-Strategien, Ressourcenkonfiguration und Account-Leitplanken — vollautomatisch.

Strategie

Setzt Governance- und Cost-Policies als Code durch — damit strategische Entscheidungen langfristig und teamübergreifend tragen.

Architektur

Prüft Netzwerk-Topologie, Datenresidenz und Sovereignty-Anforderungen gegen providerneutrale WAF++-Controls.

Standards

Wendet Zero-Trust- und Security-Controls direkt auf Terraform an — mit klaren PASS-, FAIL- und SKIP-Ergebnissen für jeden Check.

Was ist WAFPass?

WAFPass liest deine Terraform-.tf-Dateien und wertet sie gegen maschinenlesbare YAML-Control-Definitionen aus — für sofortiges, strukturiertes Feedback, ob deine Infrastruktur den WAF++-Standards entspricht. Kein Cloud-Zugriff nötig. Kein Laufzeit-State erforderlich.

Statische Analyse — funktioniert vor terraform apply
Providerneutrale Controls über alle sieben Säulen
Native CI/CD-Integration — GitHub Actions, GitLab CI, Pre-commit

Drei klare Ergebnisse

Jeder Control-Check liefert eines von drei Ergebnissen — keine Mehrdeutigkeit, kein versteckter State.

  • PASS – Control-Anforderungen vollständig erfüllt
  • FAIL – Anforderungen nicht erfüllt, mit Remediation-Hinweis
  • SKIP – Erfordert Laufzeit-Kontext, zur manuellen Prüfung markiert

WAFPass in der Praxis

Ein einzelner Befehl liefert dir einen strukturierten Compliance-Report über deine gesamte Terraform-Codebasis — Control für Control, Säule für Säule, mit klarem Remediation-Hinweis für jeden Fehler.

WAFPass erzeugt einen PDF-Report, den du im Team teilen, an ein ADR anhängen oder direkt an Auditoren weitergeben kannst — ohne zusätzliches Tooling.

PASS / FAIL / SKIP pro Control
Findings nach Schweregrad sortiert
Teilbarer PDF-Output

Gebaut für echte Engineering-Workflows

WAFPass ist dafür gedacht, in deiner Pipeline zu leben — nicht als einmaliges Audit-Tool, sondern als kontinuierliches Compliance-Gate.

Automatische Validierung

Über 20 Assertion-Operatoren werten Terraform-Ressourcen gegen Controls aus — vollautomatisch für statische Analyse, klar markiert für Runtime-Checks.

Säulen-Filterung

Führe Checks für eine einzelne Säule, eine Menge von Control-IDs oder einen bestimmten Schweregrad aus — fokussiert auf das, was im jeweiligen Kontext zählt.

CI/CD-Integration

Native Unterstützung für GitHub Actions, GitLab CI und Pre-commit-Hooks. Konfigurierbare Exit-Codes machen Pipeline-Gating unkompliziert.

Remediation-Hinweise

Jedes FAIL enthält klare Hinweise, was geändert werden muss — kein Raten, kein manuelles Quersuchen in der Dokumentation.

YAML-Controls

Controls sind maschinenlesbares YAML — versioniert, erweiterbar und teilbar. Eigene Controls mitbringen oder die community-gepflegte WAF++-Control-Bibliothek nutzen.

Alle sieben Säulen

Security, Cost, Performance, Reliability, Operations, Sustainability, Sovereignty — ein Tool deckt das gesamte WAF++ Framework ab.

Wie WAFPass funktioniert

WAFPass folgt einem einfachen, vorhersehbaren Ablauf — von deinen Terraform-Dateien zu einem strukturierten Compliance-Report in Sekunden.

  1. Terraform-Pfad angeben
    wafpass check <PFAD> gegen eine Datei oder ein Verzeichnis mit .tf-Konfigurationen ausführen.
  2. Controls werden ausgewertet
    WAFPass parst HCL2 und wendet YAML-Control-Assertions an — gefiltert nach Säule, ID oder Schweregrad.
  3. Ergebnisse mit Remediation
    Jeder Control liefert PASS, FAIL oder SKIP — mit Remediation-Hinweis für jeden Fehler, zusammengefasst im vollständigen Report.

Aktuell in der Beta

WAFPass benötigt Python 3.10+ und deine vorhandenen Terraform-Dateien. Keine Cloud-Credentials, kein Laufzeit-State — nur dein Code und die WAF++-Controls.

WAFPass befindet sich in aktiver Entwicklung. Die PyPI-Verfügbarkeit ist für das v1-Release geplant. Folge dem Projekt für Updates.
