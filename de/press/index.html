TEXT

Freigegebener Boilerplate-Text

Folgender Text kann bei der Beschreibung von WAF++ in Artikeln, Podcasts oder Präsentationen verwendet werden.

Kurz (1 Satz)

WAF++ ist ein Open-Source-, anbieterneutrales Framework zur Bewertung und Verbesserung der Qualität von Cloud-Workloads über 8 Säulen: Sicherheit, Zuverlässigkeit, Performance-Effizienz, Kostenoptimierung, operationelle Exzellenz, Nachhaltigkeit, Souveränität und Agentic.

Social / Kurzform (≤ 280 Zeichen)

WAF++ ist ein Open-Source-, anbieterneutrales Framework für Cloud-Architekturqualität — 8 Säulen, transparentes Scoring, community-gesteuert via öffentliche RFCs. Doku: CC BY 4.0, Code: Apache 2.0. waf2p.dev

Standard (2–3 Sätze)

WAF++ (Well-Architected Framework Plus Plus) ist ein Open-Source-, community-gesteuertes Framework zur Bewertung und Verbesserung der Qualität von Cloud-Workloads. Es umfasst 8 Säulen — Sicherheit, Zuverlässigkeit, Performance-Effizienz, Kostenoptimierung, operationelle Exzellenz, Nachhaltigkeit, Souveränität und Agentic — mit einem transparenten Scoring-Modell namens PASS.

Im Gegensatz zu proprietären Anbieter-Frameworks wird WAF++ offen über RFCs in GitHub Discussions gesteuert, sodass jede Entscheidung nachvollziehbar und jeder Control zitierbar ist. Das Framework ist kostenlos nutzbar, kann frei beigetragen und weiterentwickelt werden.

Podcast / Interview-Intro

WAF++ — ausgesprochen „waf plus plus" — ist ein Open-Source-Framework, das Engineering- und Architektur-Teams eine anbieterneutrale Linse für die Bewertung von Cloud-Workloads gibt. Es strukturiert alles entlang von acht Säulen: Sicherheit, Zuverlässigkeit, Performance-Effizienz, Kostenoptimierung, operationelle Exzellenz, Nachhaltigkeit, Souveränität und Agentic.

Was WAF++ besonders macht, ist die Art, wie es entsteht: Jede Regel, jedes Scoring-Kriterium und jede Designentscheidung durchläuft einen öffentlichen RFC-Prozess auf GitHub — vollständig transparent, vollständig community-getrieben. Kein Vendor dahinter, kein Upsell, keine Black Box. Jede:r kann die Kriterien lesen, hinterfragen und verbessern.
