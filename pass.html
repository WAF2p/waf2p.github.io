WAFPass

WAFPass is the official CLI for the WAF++ Framework — an automated compliance checker that validates your Terraform infrastructure against the seven pillars: security, cost, performance, reliability, operations, sustainability, and sovereignty.

Documentation

PASS – Platform · Architecture · Strategy · Standards

WAFPass applies four PASS perspectives as automated checks against your infrastructure — making compliance decisions traceable, repeatable, and auditable across any cloud.

Platform

Validates baseline platform controls — tagging strategies, resource configuration, and account-level guardrails — automatically.

Strategy

Enforces governance and cost policies as code — so strategic decisions hold over time and across teams.

Architecture

Checks network topology, data residency, and sovereignty requirements against provider-neutral WAF++ controls.

Standards

Applies zero-trust and security controls directly to Terraform — with clear PASS, FAIL, and SKIP outcomes for every check.

What is WAFPass?

WAFPass reads your Terraform .tf files and evaluates them against machine-readable YAML control definitions — giving you immediate, structured feedback on whether your infrastructure meets WAF++ standards. No cloud access needed. No runtime state required.

Static analysis — works before terraform apply
Vendor-neutral controls across all seven pillars
Native CI/CD integration — GitHub Actions, GitLab CI, pre-commit

Three clear outcomes

Every control check produces one of three results — no ambiguity, no hidden state.

  • PASS – control requirements fully met
  • FAIL – requirements not met, with remediation guidance
  • SKIP – requires runtime context, flagged for manual review

How WAFPass works

WAFPass follows a simple, predictable execution flow — from your Terraform files to a structured compliance report in seconds.

  1. Point at your Terraform
    Run wafpass check <PATH> against a file or directory of .tf configurations.
  2. Controls are evaluated
    WAFPass parses HCL2 and applies YAML control assertions — filtering by pillar, ID, or severity as needed.
  3. Results with remediation
    Every control produces a PASS, FAIL, or SKIP — with remediation guidance for every failure, aggregated into a full report.

Currently in Beta

WAFPass requires Python 3.10+ and your existing Terraform files. No cloud credentials, no runtime state — just your code and the WAF++ controls.

WAFPass is in active development. PyPI availability is planned for the v1 release. Follow the project for updates.
